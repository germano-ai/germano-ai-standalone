1. Come si usa (Primi Passi)

Germano AI Standalone è stato progettato per essere "Plug & Play". Non richiede installazione, server web o configurazioni di sistema complesse.

  1. Scarica l'archivio ZIP dal link in homepage e scompattalo in una cartella a tua scelta.
  2. Fai doppio clic sul file START.bat (su Windows). Questo avvierà il motore in background e aprirà automaticamente la Dashboard nel tuo browser.
  3. Copia i tuoi documenti da anonimizzare dentro la cartella input/.
  4. Dalla Dashboard, assicurati di aver selezionato il "Profilo di Conformità" desiderato (PA o PD) e clicca su START DAEMON.
  5. Attendi l'elaborazione. Troverai i file puliti nella cartella output/.

2. Integrazione API REST

Germano AI Standalone include un endpoint API locale che ti permette di inviare documenti in modo programmatico dal tuo software (es. Gestionale, CRM o Portale Web PA) e ricevere immediatamente il file anonimizzato in risposta.

Endpoint

POST http://127.0.0.1:8000/api/v1/anonymize

Esempio di Richiesta (cURL)

curl -X POST http://127.0.0.1:8000/api/v1/anonymize \ -F "file=@/percorso/al/tuo/documento.pdf"

Risposta

L'API è sincrona. Attenderà l'elaborazione del demone IA in background (fino a 60 secondi) e restituirà direttamente il flusso binario del file anonimizzato come allegato scaricabile.

  • 200 OK: File elaborato con successo.
  • 400 Bad Request: Nessun file inviato.
  • 503 Service Unavailable: Il demone IA è spento, avvialo prima dalla Dashboard.
  • 504 Gateway Timeout: L'elaborazione ha richiesto troppo tempo (es. file troppo grande o GPU assente).

3. Struttura delle Cartelle

Il sistema organizza il lavoro automaticamente spostando i file in base al loro stato:

  • /input: La cartella di partenza. Inserisci qui i file. Il sistema supporta anche sottocartelle (che verranno replicate fedelmente nell'output).
  • /output: Dove finiscono i documenti anonimizzati. Se il file era un DOCX, troverai un DOCX, formattato e impaginato come l'originale.
  • /processed: Dove vengono parcheggiati i file originali dopo essere stati elaborati, per evitare di sovrascriverli o perderli.
  • /failed: Se un documento è protetto da password, corrotto, o genera errori imprevisti, viene spostato qui per l'analisi manuale.
  • /extracted: Contiene un file .json per ogni documento processato, con la mappa esatta (log) di cosa è stato trovato e sostituito (utile a fini di audit e controllo).

4. Tecnologie Utilizzate

Dietro le quinte, Germano AI orchestra i migliori modelli Open Source disponibili per garantire massima affidabilità:

  • Transformers (Hugging Face): Utilizza il modello NLP `openai/privacy-filter` per la "Named Entity Recognition" (NER), per intercettare nomi di persona e luoghi all'interno del contesto di una frase.
  • Espressioni Regolari (RegEx): Un solido dizionario di RegEx italiane per catturare inequivocabilmente Codici Fiscali, Partite IVA, Numeri di Telefono e Indirizzi Email.
  • EasyOCR: L'Optical Character Recognition entra in azione quando elabori scansioni, PDF immagini o foto di documenti d'identità. Legge il testo "pixelato" per permettere ai motori NLP di censurarlo disegnando dei rettangoli neri.
  • OpenCV: Rilevamento facciale biometrico automatico tramite classificatori di Haar per l'oscuramento visivo delle fototessere sui documenti.
  • PyTorch con DirectML: Consente di accelerare l'AI non solo su GPU Nvidia (CUDA), ma anche su GPU AMD e Intel.

5. Requisiti di Sistema Hardware

Poiché il sistema esegue l'Intelligenza Artificiale localmente, richiede potenza di calcolo. Un PC lento riuscirà comunque ad anonimizzare i file, ma impiegherà più tempo.

Minimi:
- CPU: Intel Core i5 / AMD Ryzen 5 (di 8° generazione o superiore)
- RAM: 8 GB
- OS: Windows 10/11 64-bit
- Spazio: 5 GB liberi su disco
Consigliati (per elaborazioni massive):
- CPU: Intel Core i7 / AMD Ryzen 7
- RAM: 16 GB o più
- GPU: Scheda Video Dedicata (Nvidia RTX, AMD Radeon) con almeno 4GB di VRAM
- Storage: SSD NVMe
